# CTA & Button Wording Guide (v1.1)

**Core rule:** A button label is a promise.

## 1. Default Pattern
Verb + Noun  
Drop articles. Use sentence case.

Examples:
- Add job
- Create report
- Save changes
- Delete user

## 2. Verb Selection
- Create → new entity
- Add → attach existing
- Save → persist edits
- Submit → enter workflow
- Send → transmit
- Delete → permanent
- Remove → detach

Avoid: OK, Yes/No, Click here, Done, Continue*

## 3. Length & Tone
- 1–3 words preferred
- 4 max
- No punctuation, emojis, fluff, or politeness

## 4. Hierarchy
- One primary action per view
- Never two primaries
- Destructive actions repeat verb + object

## 5. Icons
- Text first, icon optional
- Icon-only only for universal actions
- Icon-only requires tooltip + aria-label

## 6. Accessibility
- <button> for actions, <a> for navigation
- Min target 44×44px
- aria-label when ambiguous

## 7. Quick Check
1. Starts with verb?
2. Outcome predictable?
3. Only one primary?
