# Claude Code Global Instructions

## External Rules Source
This machine keeps canonical agent rules in `~/.agents/rules/`.

When starting work:
1. Treat files in `~/.agents/rules/` as active global guidance.
2. Prefer these rules when they add constraints or style guidance.
3. If a repository has local instructions, merge them with these global rules.
4. If instructions conflict, follow this precedence:
   - system/developer instructions
   - repository-local instructions (e.g. `AGENTS.md`)
   - `~/.agents/rules/*`

## Rule Application Policy
These files are design/copy guidance, not hard runtime constraints.

Apply `~/.agents/rules/*` by default when work includes:
- UI layout, interaction patterns, components, or visual styling
- UX writing, labels, CTA/button text, form content, and table presentation
- Design system or frontend implementation decisions

Do not force these rules when work is primarily:
- backend logic, infrastructure, build tooling, tests, or data migrations
- API correctness, security, or performance work with no user-facing design/copy impact

If a rule would reduce correctness, accessibility, security, or maintainability, prioritize those qualities and treat the design rule as advisory.

When multiple rules apply:
1. Use the most specific rule for the element (e.g. CTA guide for buttons, form guide for forms).
2. Use broader documents (e.g. `functionalist-design.md`) as tie-breakers.
3. If still ambiguous, choose the option that is most predictable and easiest for users to complete tasks.

## Symlinked Rules Mirror
A mirrored rules folder exists at `~/.claude/rules/` and should contain symlinks to `~/.agents/rules/*`.
Always resolve symlinks and read the source files.

## Sync Command
If instructions or rules are added or changed, run:

```bash
~/.agents/sync.sh
```

This copies `~/.agents/claude/CLAUDE.md` to `~/.claude/CLAUDE.md` and refreshes rule symlinks.

## Attribution Policy
Never attribute work to Claude in repository artifacts.

For any PR, commit message, or review/discussion comment:
- Do not mention Claude, Claude Code, Anthropic, or "AI assistant" authorship.
- Do not add signatures or footers such as "Generated by Claude" or "Co-Authored-By: Claude".
- Write in a neutral teammate voice with no model/tool self-reference.
