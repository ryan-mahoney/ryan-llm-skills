# Global Codex Instructions

## Instruction Type Clarification
The files in `~/.agents/rules/*.md` are design and UX guidance documents.
For Codex, treat them as instruction context (not `.rules` command-approval policies).

Use Codex `.rules` files only for command approval behavior (for example `~/.codex/rules/*.rules`).

## Canonical Design Guidance Source
Canonical source: `~/.agents/rules/`
Symlinked mirror for convenience: `~/.codex/guides/`

When relevant, load and apply guidance from these files:
- `cta-design.md`
- `form-design.md`
- `table-row-design.md`
- `functionalist-design.md`

## When To Apply
Apply this guidance by default for:
- frontend UI layout and interaction design
- UX copy and button/CTA wording
- form and table presentation decisions
- design system and component-level styling decisions

Do not force this guidance for:
- backend, infra, CI/CD, build tooling, migrations, or pure API logic
- tasks with no user-facing design or copy impact

If guidance conflicts with correctness, security, accessibility, or maintainability, prioritize those qualities.

## Instruction Precedence
Follow normal Codex instruction precedence from global to local scope, with deeper project files overriding broader ones.
Within this global file, apply design guides as advisory context.

## Sync Command
If instructions or rules are added or changed, run:

```bash
~/.agents/sync.sh
```

This copies `~/.agents/codex/AGENTS.md` to `~/.codex/AGENTS.md` and refreshes guide symlinks.

## Attribution Policy
Never attribute work to Codex in repository artifacts.

For any PR, commit message, or review/discussion comment:
- Do not mention Codex, OpenAI, GPT, or "AI assistant" authorship.
- Do not add signatures or footers such as "Generated by Codex" or AI co-author lines.
- Write in a neutral teammate voice with no model/tool self-reference.
